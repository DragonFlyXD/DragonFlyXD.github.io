【Javascript】检测变量与对象的类型

发表于 2017-03-28 | 分类于 Javascript
字数统计 683 | 阅读时长 3

首先，我们知道js的数据类型分类两种，如下所示：

  1. 基本类型：Undefinded,Boolean,Number,String,Null
  2. 引用类型：Array,Date,RegExp,Function,Promise，而它们又继承于Object,因此Object为它们的基类

然有了数据类型，我们自然要判断它们的所属类型，常见的检测方法如下：

  • typeof操作符
  • instanceof操作符

以上两种检测方法在各自检测领域都十分有效，但如果我想将检测范围扩至囊括所有类型或者我只想检测纯粹的Object类型呢？

细想，引用类型皆为Object类型，而检测null的结果也为Object，然而当我们想检测对象类型时，怎么分辨它是何种类型？这时候，就该Object.prototype.toString.call()“粉墨登场”了。

Object.prototype.toString.call() || {}.toString.call()

最后，我们来分析一下Object.prototype.toString.call(obj)这个方法的结构：

  • Object.prototype.toString为调用Object的toString方法,能返回反映这个对象的字符串。而其他类型的toString皆重写了Object的toString方法根据原型链的知识，如obj.toString()时调用的是重写的toString()方法，所以得不到其对象类型。
  • call()方法用来改变函数运行的上下文，具体点就是改变函数运行时this的指向。因此我们可以借助call来得知obj的具体类型。
