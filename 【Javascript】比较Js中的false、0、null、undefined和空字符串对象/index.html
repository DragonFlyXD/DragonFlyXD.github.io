【Javascript】比较Js中的false、0、null、undefined和空字符串对象

发表于 2017-03-28 | 分类于 Javascript
字数统计 507 | 阅读时长 2

类型：

运行结果均为true。也就是说，false为布尔类型，0为数字类型，null为object类型，空字符串为字符串类型，undefined为undefined类型。

互等性：

运行结果如上所示，得出0和空字符串与false相等，null和undefined与false不相等，而null与undefined相等。

因此，我们把0、””、false归为一类，称之“假值”;而把null和undefined归为一类，称之“空值”。

运行后，false、””、0皆可运行，而null与undefined则会报错。

因为假值还算是个有效的对象，能对其使用toStirng()等类型相关的方法，而空值则不行。

字符串表示：

运行结果如上，虽然null和undefined不能使用toString()方法，但可以用String构造函数进行构造。

注意,如果对空值使用decodeURI()这样的函数，返回的是”null”、”undefined”字符串。

空值和假值作为if分支条件：

运行后，控制台无值，这是因为它们都在各自类型中被视为无效值或空值，所以在if条件中都被当做false来对待。

null和undefined的区别：

  • undefined和null对象无非是两个特殊对象，undefined表示无效对象，null表示空对象。
  • 如果变量显式或者隐式（由Javascript引擎进行赋值）地被赋予了undefined，那么代表了此变量未被定义，如果被赋予null值，则代表此变量被初始化为空值。
  • 变量如果声明了但是没有初始化，那么Javascript引擎会将此变量自动指向undefined对象。
