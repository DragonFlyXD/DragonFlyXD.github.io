【Javascript】部分容易忘却的基础

发表于 2017-03-28 | 分类于 Javascript
字数统计 1,580 | 阅读时长 6

作用域

js代码自上而下执行。
但是，js代码在整体运行时分：
词法分析期与运行期。
js代码自上而下执行之前，现有一个“词法分析过程”！

词法分析

  1. 先分析参数
  2. 再分析变量声明
  3. 最后分析函数声明

具体步骤：

  1. 函数运行前的一瞬间，生成Activation Object(活动对象),下称AO。
  2. 函数声明的参数,形成AO的属性，值全是undefined;接收实参，形成AO相应的属性的值。
  3. 分析变量的声明，如var age;如果AO上还没有age属性，则添加AO属性，值为undefined；如果有，则不做任何影响。
  4. 分析函数的声明，如function foo(){},则把函数赋给AO.foo属性，如果此前foo属性已存在，则被覆盖。

注意：
function t1(){}(函数声明) 与 t1=function(){}(函数表达式)

这两种函数的定义方式在词法分析期时，有着本质的区别。前者在词法分析期就发挥作用，而后者在运行期才发挥作用。

此外，作用域链是指由函数由内到外，产生的AO链([[Scope]])。

执行函数时会创建一个称为“运行期上下文(execution context)”的内部对象，运行期上下文定义了函数执行时的环境。

arguments

定义：arguments是函数运行时的实参列表,是类数组对象。

arguments.callee：该属性是一个指针，指向拥有这个arguments对象的函数。

闭包

定义：是指有权访问另一个函数作用域中的变量的函数。

例子：

分析：在c语言中,t1被调用执行，则申请内存，并把它局部变量push入栈。执行完毕后，t1中的局部变量随着函数的退出而销毁。

而在JS中，t1执行过程中又产生了t2,而从作用域上来说，t2能访问到age=20。于是，“age=20”并没有消失，而是与返回的t1函数形成了一个“环境包”，这个包属于t2,称之“闭包”。

this

4种用法:

1.作为普通函数调用时，this的值指向window。准确来说，this的值为null，被解释成了window。

2.作为对象的方法来调用，this指向方法的调用者，即该对象。

3.函数作为构造函数调用时。因为js里没有类的概念，创建对象是用构造函数来完成的，或者直接用json格式{}来写对象。

运行具体步骤：

  1. 系统创建了空对象{}，空对象constructor属性指向了Car函数。
  2. 把函数的this指向了该空对象。
  3. 执行该函数
  4. 返回该对象

4.函数通过call，apply调用。

语法格式：fn.call(对象obj,参数1，参数2,…参数N);fn.apply(对象obj,[参数1，参数2,…参数N])。

运行如下：

  1. fn函数中的this属性指向对象obj。
  2. 运行fn(参数1，参数2，…参数N)。

call与apply区别:

  • apply和call两者在作用上是相同的，但两者在参数上有区别的。
  • 对于第一个参数意义都一样，但对第二个参数：

apply传入的是一个参数数组，也就是将多个参数组合成为一个数组传入，而call则作为call的参数传入（从第二个参数开始）。
如 fn.call(对象obj,参数1，参数2,…参数N)对应的apply写法为：fn.apply(对象obj,[参数1，参数2,…参数N])。

  • 同时使用apply的好处是可以直接将当前函数的arguments对象作为apply的第二个参数传入。

面对对象

对象的特点

  • 在js中，有对象，但没有类(有构造函数)。因此，js中的对象可以不依赖类而存在。

  • js中的对象，就是”一组属性和值的集合”,类似于PHP中的关联数组。

  • js中的对象的属性和方法不必区分。

  • js中的对象的属性是可以任意增添和删除的。

私有属性与封装

原理：用闭包来完成js面对对象的私有属性。

要求：不让外界直接访问Person的name属性。

原型链继承

起因：在js里没有类的概念，因此js的继承不是通过类来实现的，而是通过”原型”的概念来完成的。

过程：

分析：

  1. 构造空对象{}。

  2. tiger.run=function(){}。

  3. tiger._proto_=Tiger.prototype。(这是继承的关键)

分析：

Animal的prototype属性指向默认原型对象(实质为空对象)，而默认原型对象的constructor属性又指向Animal,所以默认原型对象表现为Animal。

注意：

  • js里的每个对象都有一个__proto__属性指向其原型对象。
  • 通过对象字面量构造的对象，其prototype属性指向object.prototype。而通过构造函数new出来的对象，其prototype属性指向其构造函数的prototype所指向的对象。
  • _proto_(隐式原型)与prototype(显式原型)的关系：隐式原型指向创建这个对象的函数(constructor)的prototype。
  • prototype属性的指向路径：Animal—>默认原型对象—>object—>null(原型链)
  • 对象的属性和方法就是沿着原型链来查找和调用的。
