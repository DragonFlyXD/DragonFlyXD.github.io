【Laravel】bcrypt加密与Postman接收四舍五入后一位小数的问题

发表于 2017-03-28 | 分类于 Laravel
字数统计 315 | 阅读时长 1

你可能遇见一个常见的场景，当用户注册，你使用bcrypt加密了密码。但当用户修改密码时你使用用户填写的密码加密后与数据库中的该用户的加密后的密码相校验，发现就算它们加密前的密码是相同，但无论怎么比较都会报错。

由于Bcrypt的加密系数(word fator)可以任意调整，使其每一次加密的次数可以随着硬件设备的的升级而增加。

解决方法

使用 Laravel中的 Hash facade 的 check 方法来校验密码的正确性。

做api开发的时候，Postman是利器。今日遇见一个情况，使用Postman接收一个后端传过来的四舍五入后一位的数字时，会显示该数字四舍五入之前的原样。

在Postman中依旧会显示2.677777777。但在前台页面中接收的值的确是2.7。
