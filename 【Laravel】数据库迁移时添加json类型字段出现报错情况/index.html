【Laravel】数据库迁移时添加json类型字段出现报错情况

发表于 2017-03-28 | 分类于 Laravel
字数统计 401 | 阅读时长 2

数据库迁移时添加json类型字段出现报错情况,如下所示：

最可能原因：mysql版本过低(如我的为mysql5.5)，而json类型的字段在>=mysql5.7版本上才能使用。

更新mysql版本即可，如我将mysql版本更新至8.0。(注意，mysql版本从5.5、5.6到5.7之后的版本为8.0，因此可以将5.6和5.7理解为6.x，7.x)

mysql版本升级方法(详见博文)。

如果mysql在一定的集成环境中，如wamp则可以前往官网下载mysql的addon，而mamp则要进行版本升级。

以下杂谈：

  • MAMP的mysql命令行位置：

/Applications/MAMP/Library/bin/mysql

  • 在数据库迁移时添加唯一索引引发1071错误：

mysql建立索引时，计算key的长度是所有索引用到的字段的char长度再乘以一定的倍数，且不能超过767字节。(innoDB单列索引长度不能超过767字节,复合索引长度不得超过3072，MyISAM单列索引和复合索引均不能超过1000字节)

该倍数会根据数据的字符集而定，例如：

latin1 = 1 byte = 1 character
uft8 = 3 byte = 1 character
gbk = 2 byte = 1 character

所以，修改下字段长度即可。

或者配置迁移生成的默认字符串长度。

修改位置在app\Providers\AppseriviceProvider.php。
