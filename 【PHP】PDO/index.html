【PHP】PDO

发表于 2017-03-28 | 分类于 PHP
字数统计 724 | 阅读时长 3

Introduction

  • PHP数据对象(PHP Data Objects)扩展为PHP访问数据库定义了一个轻量级的一致接口。
  • 实现了PDO接口的每个数据库驱动可以公开具体数据库特性作为标准扩展功能，也就是说实现了PDO接口后不仅能访问mysql一种数据库,其他数据库亦能。
  • PDO提供了一个数据访问抽象层，这意味着，不管使用那种数据库，都可以用相同的函数(方法)来查询和获取数据。另外，PDO不提供数据库抽象层。

三个类

  • PDO类，与数据库连接有关(连接、执行sql语句)。
  • PDOStatement 处理结果集。
  • PDOException 异常处理类。

获取PDO实例

  • PDO::__construct ( string $dsn [, string $username [, string $password [, array $driver_options ]]] )。
  • dsn(data source name)即数据源名称，包含了请求连接到数据库的信息。已连接mysql为例:mysql:dbname=testdb;port=3306;host=127.0.0.1。
  • driver_options为一个具体驱动的连接选项的键=>值数组。

执行sql语句的部分方法

  • public PDOStatement PDO::query ( string $statement),执行一条sql语句，并返回一个为PDOStatement对象的结果集。如果失败，返回false。
  • public PDOStatement PDO::prepare ( string $statement [, array $driver_options = array() ] ),负责要准备执行的查询，并返回一个PDOStatement对象。如果失败，返回false或者抛出一个PDOException错误。
  • int PDO::exec ( string $statement),执行一条sql语句，返回受修改或删除sql影响的行数。如果没有，则返回0。可能返回false或者等于与false的非布尔值。
  • bool PDOStatement::execute ([ array $input_parameters ] ) 执行一条预处理语句，成功是返回true，失败时返回false。
  • bindParam与bindValue方法的不同是，前者绑定的参数只能为变量，而后者绑定的参数可以为具体的值。

NOTE

  • 使用prepare方法时，对于每次执行迭代中要改变的列值，必须使用占位符而不是具体的列值。占位符有两种语法，命名参数和问号参数，
  • 命名参数，关联数组按名称使用，和顺序无关，例如 INSERT INTO student SET name=:DragonFly。
  • 问号参数，索引数组，按索引顺序使用，例如 INSERT INTO student SET name=?;

Summary

  • 每次将查询发送给mysql服务器时，都必须解析该查询的语法，确保结构正确并能够执行。如果只发送一次，影响不大。倘若反复执行相同的查询，额外带来的开销会很大。
  • 预处理语句会在服务器上缓存查询的语法和执行过程，而只在服务器和客户端之间传输有变化的值，以此来减少额外的开销。
  • 如果只在程序中使用一次select语句，可以考虑使用query方法。如果需要多次发送，可以考虑使用prepare方法返回一个PDOStatement对象后使用execute发送语句。
