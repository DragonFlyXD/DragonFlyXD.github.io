符号->、=>和::三者的区别

定义：

  • ->,插入式解引用操作符(infix dereference operator),将对象的引用指向对象，例如对象->对象的方法。
  • =>,一般用于数组，键名=>键值。
  • ::,作用域解析操作符(Scope Resolution operator),用于调用类的静态成员变量或是类之间的调用。

this、self、parent三者的区别

定义：

  • this是指向当前对象的指针。
  • self是指向当前类的指针。
  • parent是指向父类的指针。

例子：

this

运行后，发现this第一次指向实例后的p1,第二次指向实例后的p2。

因此，this是指向当前对象的指针，而不指向其他对象或类。

self

self指向类的本身，而与实例化的对象不相关，且self一般用来调用类的静态成员。

parent

self是指向父类的指针，一般用来调用父类的构造函数。

isset、is_null、empty三者的区别

素材

  • $a:没定义。
  • $b:有定义，但没初始化。
  • $c:有定义，赋值为1。
  • $d:有定义，赋值为null。

is_null

empty

Description

  • empty、asset首先都会检查变量是否存在，然后对变量值检测。而is_null只是直接检查变量值，是否为null，因而如果变量为定义就会有警告出现。
  • 在php5.50版本之前，empty和asset仅支持变量(以$字符开头的)，而is_null输入参数只要是能够有返回值就行。
  • empty,如果变量是非空或非零的值，则返回false，否则返回true。换句话说，””、0、0.0、”0”、NULL、FALSE、array()、$var(声明了，但未负值)都会被认为是空的，返回true。
  • isset,如果变量存在且不为null，则返回true，否则返回false。
  • is_null,检测传入值是否为null，则返回true，其他都返回false。如果一个变量定义了未赋值，会报出一个没有定义变量XXX的警告。

pathinfo、getFilename、getImageFilename三者的区别

Description

  • getFilename函数已经过时，而getImageFilename作为其代替出场。如果使用它，只会返回一个空字符串。
  • getImageFilename会返回包含图片绝对路径的字符串，失败则抛出imagickException错误。
  • pathinfo则是解析一个关于文件路径的字符串，并返回一个关于其文件信息的关联数组。mixed pathinfo ( string $path [, int $options = PATHINFO_DIRNAME | PATHINFO_BASENAME | PATHINFO_EXTENSION | PATHINFO_FILENAME ] )，第二个options是可选项，如果不做选择，其路径信息全部输出。

Example

NOTE

  • pathinfo函数只是解析你的路径信息，不管你路径写的是否对错。
  • 例如 var_dump(pathinfo('./images/齿轮..png',PATHINFO_BASENAME));,它会输出string(11) “齿轮..png”，因此，输入参数要谨慎点。

魔法常量__DIR__、__FILE__、和函数dirname三者的区别

Description

  • __DIR__,返回文件所在的目录，也就是当前文件的父路径，除非是跟目录，否则目录名中不包含末尾的斜杠。
  • __FILE__,返回文件的完整路径和文件名，也就是一个包含当前文件名的绝对路径。
  • dirname,返回目录中的目录部分，换句话说，dirname(__FILE__)等价__DIR__。
