【PHP】升级mac自带的php后MAMP的apache不能启动

发表于 2017-03-28 | 分类于 PHP
字数统计 598 | 阅读时长 2

真是天坑啊~~~~为了这个问题弄了我一下午，记录一下，希望我下次不要再犯(好像有一次也是因为这个问题?)

原因是：升级了Mac自带的php版本后，它会重启系统自带的apache，因此启动不了MAMP上的apache，要先sudo apachectl stop掉系统自带的apache才行。

以下内容是我个人的聊骚。

因为YII2框架的体系，做前后端分离时，与Vue的契合搞得我很是头疼，所有准备学习下据说很优雅的Laravel(5.3版本默认的前段框架为Vue.js)。然而5.3版本的php要求>=5.6.4,而我Mac上的版本为5.5，所以通过composer下过来的的laravel限定成5.2版本，因此我准备更新下系统自带的php版本。

另一种方法是修改系统的配置文件，把MAMP上的php版本绑定上去。只不过我顽强的选择更新(作死/(ㄒoㄒ)/~~)。

我通过curl去升级php版本(使用brew亦行)，等了漫长的20分钟，将php版本升级后,修改了.bash_profile文件，终端里输入php -v显示版本为php7.0。然后我满怀欢喜的下了5.3版本的laravel，启动MAMP,发现左上角的绿灯只有mysql亮了，而apache的灯没亮，脊背一凉，随即弹出apache不能正常启动，要我去看apache错误日志，从这时起我便踏上了折腾之路。

以下我谈谈MAMP上的apache不能启动的个别原因：

  • 可能是Mac版本在10.10以上时，在/Applications/MAMP/Library/bin中里envvars文件命名问题。通过以下几条命令可以解决。

    cd /Applications/MAMP/Library/bin (跳转到bin目录下)

    mv envvars _envvars (将envvars重名为_envvars)

  • apache端口被占,例如我选择的apache端口为80。先lsof -i:80查看被占端口的PID(进程ID)，再通过PID kill -9(9号信号是SIGKILL,强制杀死这个进程，可以通过kill -l查看所有信号) PID杀死这个进程。

  • 系统自带的apache已经开启，因此无论你怎么摆弄MAMP套件里的apache都没卵用。输入sudo apachectl stop停止掉系统自带的apache，然后在重启MAMP即可。

Summary
