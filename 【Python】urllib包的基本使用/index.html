【Python】urllib包的基本使用

发表于 2017-04-04 | 分类于 Python
字数统计 942 | 阅读时长 4

urllib是python内置的一个工具包，包含用户处理Url的几个模块。

  • urllib.request 用户打开和读写urls
  • urllib.error 包含了由urllib.request引起的异常
  • urllib.parse 用于解析url
  • urllib.robotparser 分析robots.txt文件

python3将urllib和urllib2合并为urllib，因此大体使用方法和python2相同。

以下简述的为python3的urllib工具包的相关使用方法。

urllib.request

urllib.request 模块定义了在基本和摘要式身份验证、 重定向、 cookies等应用中打开 Url (主要是 HTTP) 的函数和类。

主要方法与类：

  • urllib.request.urlopen(url, data=None, [timeout, ]**, cafile=None, capath=None, cadefault=False, context=None*)

    打开一个指定url的网址。该方法返回一个http.client.HTTPResponse对象。

    • url 可以是字符串，亦或是urllib.request.Request对象。
    • data 请求附加的额外数据，如表单数据等。
    • timeout 请求的超时时间。默认为socket._GLOBAL_DEFAULT_TIMEOUT。

  • urllib.request.Request(url, data=None, headers={}, origin_req_host=None, unverifiable=False,method=None)

    一个URL请求的抽象类。

    • url 请求地址。
    • data 请求附加的额外数据，如表单数据等。
    • headers 请求头配置。类型为一个dict。

urllib.error

urllib.error模块定义urllib.request引发的异常的异常类。

  • urllib.error.URLErro

    该模块的基本异常类,为 OSError的子类。

    • reason,返回状态码文本描述。

  • urllib.error.HTTPError

    urllib.error.URLErro的子类。用来处理与Http相关错误。

    • code 返回的http状态码。
    • reason 返回状态码文本描述。
    • headers Http响应头信息。

urllib.reponse

urllib.request.urlopen打开地址后，返回的一个响应类。

  • geturl，返回一个响应的真实的url。

    因为请求不存在的地址，些许网站会重定向至其他网页，如404页面。

    所以获取的url可能为重定向后的url。

    也可能直接抛出错误，不进行页面的重定向，这时候就获取不到url了。

  • getcode, 获取正常返回的状态码，通常是200。

  • info, 返回获取的页面信息,通常为响应头信息。

  • read,readline, 从打开的页面中读取内容。

    若不写参数，默认读取整个页面。

以下介绍下爬虫技巧：

  • 些许网站会禁止爬虫的到访，这时候需要伪装成浏览器访问。

    在headers里添加User-Agent即可。

  • 对付防盗链

    防盗链即检查你请求头里referer站点是否为本站。

    在header里设置referer请求的站点即可。
